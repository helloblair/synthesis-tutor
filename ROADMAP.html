Project Roadmap — What's Been Built

92% Complete
All core PRD requirements satisfied
Day 1 — Project Scaffolding + Layout
Vite + React 19 + TypeScript + Tailwind CSS initialized. Two-panel split layout (45% chat / 55% workspace). iPad-optimized viewport: no-zoom, no-scroll, touch-manipulation. Nunito font loaded. Full-screen fixed layout.
Day 2 — Fraction Blocks + Drag-and-Drop
FractionPalette with 9 fraction denominations (1, 1/2, 1/3, 1/4, 1/5, 1/6, 1/8, 1/10, 1/12). Rainbow color coding. Proportional sizing (height = fraction value × 400px). dnd-kit with touch sensor. Blocks draggable from palette → workspace drop zone.
Day 3 — Combine, Split, & Smash Mechanics
Tap-to-select + tap-another-to-combine (same denominator → simplified result). Double-tap-to-split (÷2, ÷3, ÷4 menu). Visual selection feedback (blue outline). Framer Motion animations on transitions. Workspace renders blocks as proportional vertical bars.
Day 4 — Conversation Engine + Lesson Script
FSM engine with useReducer — 20+ conversation nodes in lessonScript.ts. Chat UI with tutor/student message bubbles, typing delays, auto-scroll. Student responds via ResponseButtons (multiple choice). Branching logic for correct/incorrect paths with warm, encouraging tone.
Day 5 — Integration + Full Lesson Flow
Tutor prompts trigger manipulative tasks. useManipulativeChecker validates workspace state against conditions (has_half_block, has_two_quarters, shows_third_sixths, equivalent_to_third). CheckAnswerButton bridges conversation ↔ manipulative. Full 3-phase lesson plays start-to-finish. CelebrationOverlay on completion.
Day 6 — Polish + iPad Testing
Touch refinement: -webkit-touch-callout: none, overscroll-behavior: none, user-select: none. Visual polish: consistent blue borders, rebalanced panel widths, block sizing in palette. Workspace blocks rendered as vertical bars matching palette style.
Day 7 — Demo Video + README + Deploy
README written. Deployment target: Vercel. Demo video and final iPad testing remaining.

Key Engineering Decisions

Decision Choice Reason
State managementuseReducer + ContextZero deps; FSM maps perfectly to useReducer dispatch
Conversation engineFinite State Machine (JSON)Scripted branching per PRD; each node → messages + expected action + edges
Drag librarydnd-kit over react-dndTouch-first; react-dnd's HTML5 backend has poor iPad support
Block interactionsTap-to-select, double-tap-to-splitNo hover/right-click available on iPad; touch-native gestures
Workspace validationuseManipulativeChecker hookDecoupled condition checking — tutor defines condition string, hook validates against block state
No backendClient-side onlyPer PRD: single-lesson prototype, no accounts, no persistence needed
